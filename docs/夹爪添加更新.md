# 夹爪添加更新

更新时间：2026-04-28

本文记录线轨夹爪接入 Dummy V2 后的硬件拨码规则、固件事实、GUI/仿真/数据更新，以及后续 AI 或开发者应该如何安全控制夹爪。

## 结论摘要

- 线轨夹爪电机使用 CAN 节点 ID 7。
- 电机拨码器 1 不是节点 ID，它是 CAN 终端电阻相关开关；节点 ID 只由 2、3、4 号拨码决定。
- 当前夹爪验证结果：`!HAND_O` 是打开方向，`!HAND_C` 是夹紧方向。
- 夹爪机构静摩擦较大，实测需要约 `0.70A` 才能可靠起动。
- GUI 已改为“打开 / 夹紧”两态，并和关节控件放在同一个控制区，不再使用旧的 0-30 度角度滑块。
- 软件数据中夹爪统一为二值：`0.0 = 夹紧/闭合`，`1.0 = 打开`。
- 直接控制夹爪时必须有时效保护；`!HAND_O` / `!HAND_C` 会持续电流输出，停止必须发送 `!HAND_DIS`。
- 当前线轨夹爪固件的 J6 固件角限位是 `-180° ~ 180°`，不是旧 GUI 中的 `-720° ~ 720°`。
- 当前 J6 固件按 `50:1` 计算电机步数，末端实际使用 `mini8-30`，GUI 的 J6 输出角默认按 `固件角 × 50 / 30` 显示。
- 旧 MuJoCo 碰台/基座安全检查已默认关闭；新的用户软边界在“安全边界”页单独保存和启用。

## 硬件拨码规则

电机驱动板拨码器含义如下：

| 拨码 | 含义 | 权重 |
| --- | --- | --- |
| 1 | CAN 终端电阻相关，不参与节点 ID | 无 |
| 2 | 节点 ID bit2 | 4 |
| 3 | 节点 ID bit1 | 2 |
| 4 | 节点 ID bit0 | 1 |

节点 ID 计算公式：

```text
node_id = S2 * 4 + S3 * 2 + S4 * 1
```

其中 `S2/S3/S4 = ON` 时参与加权，`OFF` 时为 0。

常用配置：

| 节点 | 用途 | S1 | S2 | S3 | S4 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| 6 | J6 电机 | 按总线终端需求 | ON | ON | OFF | 4 + 2 = 6，当前已验证正确 |
| 7 | 线轨夹爪 | 当前配置 OFF | ON | ON | ON | 4 + 2 + 1 = 7 |

注意：

- S1 不改变节点 ID。不要把 S1 当成 ID 的 bit。
- 当前夹爪按 `S1 OFF, S2 ON, S3 ON, S4 ON` 配置。
- 如果后续重新设计 CAN 总线拓扑，S1 只按“是否需要终端电阻”调整，不改变夹爪节点 ID。

## 固件与电机事实

本次主控固件使用线轨夹爪资料中的：

```text
线轨夹爪资料/bin直刷/Core-STM32F4-fw-rtx-50.bin
```

当时选择 `50` 的原因：

- 资料说明中 `Core-STM32F4-fw-rtx-50.bin` 对应任同学结构件、J6 减速比 50。
- 文件名中的 `50` 是 J6 减速比，不是夹爪减速比。

后续装配确认：

- 当前 J6 末端减速器为 `mini8-30`。
- 当前已烧录 `rtx-50` 固件，因此固件命令角和真实输出角的关系是 `输出角 = 固件角 × 50 / 30`。
- 这意味着 `J6=30°` 指令理论上应看到约 `50°` 末端输出。若一点都不动，优先排查 J6 是否收到 CAN 指令、是否使能、拨码/接线/电机相序/机械锁紧，而不是减速比换算。

线轨夹爪源码事实：

```cpp
hand2 = new StepHand(_hcan, 7);
```

```cpp
StepHand(CAN_HandleTypeDef* hcan, uint8_t id)
  : CtrlStepMotor(hcan, id, false, 8, -100, 100)
```

含义：

- 夹爪在固件里是 `hand2`。
- CAN 节点 ID 是 7。
- `StepHand` 内部使用的减速参数是 8。

本次固件已通过 ST-Link 完整写入和读回校验。后续软件开发不需要重新烧录，除非再次更换固件。

### 当前关节固件限位

线轨夹爪固件源码中的 6 轴限位如下：

| 关节 | 固件限位 |
| --- | --- |
| J1 | `-170° ~ 170°` |
| J2 | `-75° ~ 90°` |
| J3 | `35° ~ 180°` |
| J4 | `-180° ~ 180°` |
| J5 | `-120° ~ 120°` |
| J6 | `-180° ~ 180°` |

固件预设：

| 指令 | 固件函数 | 姿态 |
| --- | --- | --- |
| `!RESET` | `Resting()` | `[0, -75, 180, 0, 0, 0]` |
| `!HOME` | `Homing()` | `[0, 0, 90, 0, 0, 0]` |

GUI 当前使用这些固件事实作为滑块限位和默认位置。旧资料中出现过的 `J6 -720° ~ 720°` 不适用于当前线轨夹爪固件。

### J6 末端谐波减速器显示

J6 新增末端谐波减速器后，固件仍按 `50:1` 计算 motorJ6 电机步数。GUI 中的控制和下发仍使用固件角度，但显示和 MuJoCo 视觉同步会按以下规则换算：

```text
J6 输出角 = 固件 J6 角 × 固件J6减速比 / 实际J6减速比
```

当前固件J6减速比默认为 `50.0`，实际J6减速比默认为 `30.0`，对应当前 `mini8-30`。如果后续固件或机械减速器变化，把 GUI 左侧关节控制区或“安全边界”页中的 `J6固件比`、`实际比` 改成真实值并保存配置。

## 原始夹爪指令

线轨夹爪固件新增的 ASCII 指令：

| 指令 | 作用 |
| --- | --- |
| `!HAND_EN` | 使能夹爪电机 |
| `!HAND_DIS` | 断使能夹爪电机 |
| `!HAND_I <current>` | 设置夹爪电流，范围代码中限制到约 0-2.0 |
| `!HAND_O` | 打开方向，进入电流模式 |
| `!HAND_C` | 夹紧方向，进入电流模式 |
| `!HAND_POS <1-100>` | 位置模式，当前 GUI 不使用 |
| `!HAND_ZERO` | 源码中 `HandCalibration()` 为空，当前不能当自动校准使用 |

重要安全点：

- `!HAND_O` 和 `!HAND_C` 是电流模式，会持续输出电流。
- 原始串口控制时，任何打开/夹紧动作之后都必须发送 `!HAND_DIS`。
- 不要依赖 `!STOP` 或 `!DISABLE` 来停止夹爪，原始夹爪路径应明确发送 `!HAND_DIS`。
- 不要通过通用关节命令操作 node 7，例如不要使用 `#REBOOT 7`、`#I_LIMIT_J 7 ...` 等；部分固件逻辑对 node 7 的数组索引不安全。

## 已验证的夹爪动作参数

实测状态：

- 初始夹爪在夹紧位置。
- 机构没有卡死，但齿轮/关节较涩。
- `0.15A`、`0.30A`、`0.40A`、`0.50A` 起动力不足或只能有轻微动静。
- `0.70A / 1.00s / !HAND_O` 可以打开。
- `0.70A / 0.50s / !HAND_C` 可以夹紧。

当前软件默认参数：

| 动作 | 电流 | 时间 | 后续状态 |
| --- | --- | --- | --- |
| 打开 | `0.70A` | `1.00s` | 自动双 `!HAND_DIS` 断使能 |
| 夹紧 | `0.70A` | `0.50s` | 切到低电流保持 |
| 夹紧保持 | `0.25A` | 默认最多 `60s` | 超时自动断使能 |

这个策略的目的：

- 起动阶段用 `0.70A` 克服静摩擦。
- 夹紧后不用继续大电流顶住，降低电机和机构风险。
- 又不会在夹持过程中立刻失力，因为夹紧后会进入低电流保持。
- 保持有最大时长，避免无人看管时持续发热。

## 串口/CAN 稳定性

J5 “读取位置失败: 输入/输出错误”通常是 Linux 串口设备返回的 USB/CDC I/O 错误，不是 J5 位置数值本身报错。常见诱因是主控在运动中忙于 CAN 通信、USB 连接瞬断、供电/地线噪声、或上位机一边高频下发轨迹一边高频 `#GETJPOS` 查询。

已做的软件保护：

- 真机位置轮询从 `0.5s` 降到约 `0.8s` 一次。
- `get_position()` 不再每次读取前清空输入缓存，避免误丢响应。
- 每次运动指令发送后短暂停止位置查询，避免刚下发 `>` 指令就立刻抢串口查询。
- 仿真轨迹发送默认最小间隔从 `0.02s` 增加到 `0.10s`，减轻 J6 动作时 CAN 总线拥塞。

## 用户安全边界

新增“安全边界”页：

- `启用旧 MuJoCo 碰台/基座/工作盒检查`：默认关闭，避免当前模型边界不准时阻碍装配调试。
- `启用用户软边界拦截`：总开关，只有保存并启用后才会拦截下发动作。
- `关节软边界`：记录 6 个关节的允许角度范围。
- `末端世界坐标软边界`：记录 TCP 的世界坐标盒范围。
- `J6固件比 / 实际比`：保存 J6 输出角显示和仿真映射配置。

推荐使用方式：

1. 保持用户软边界关闭，先低速移动机械臂到空间安全极限。
2. 在“安全边界”页点击 `当前关节设为下界/上界`，或用 `扩展包含当前位置` 逐步包住允许区域。
3. 对末端空间边界，移动 TCP 到安全盒的角点，点击 `当前末端设为下角/上角`，或用 `扩展包含当前末端`。
4. 勾选需要启用的边界。
5. 点击 `保存并应用`。

边界语义：

- 固件硬限位仍由 `JOINT_LIMITS` 保护。
- 用户软边界用于限制你的工作空间，不代表机械结构极限。
- 关节边界和世界坐标边界可以独立启用。
- 未启用用户软边界时，GUI 不会因为这些配置拦截运动。

## 软件控制入口

优先使用 `DummyRobot` 的高级接口，不要在业务逻辑里直接拼原始串口指令。

推荐入口：

```python
from core.gripper import GRIPPER_OPEN, GRIPPER_CLOSED

robot.set_gripper_state(GRIPPER_OPEN)    # 打开
robot.set_gripper_state(GRIPPER_CLOSED)  # 夹紧
robot.disable_gripper()                  # 明确断使能
```

参数配置：

```python
robot.configure_gripper(
    open_current_a=0.70,
    open_pulse_s=1.00,
    close_current_a=0.70,
    close_pulse_s=0.50,
    hold_current_a=0.25,
    hold_after_close=True,
    max_hold_s=60.0,
)
```

真实硬件控制前必须关闭占位模式：

```python
robot.set_gripper_placeholder_mode(False)
```

占位模式下可以安全记录 AI 数据：

```python
robot.set_gripper_placeholder_mode(True)
robot.set_gripper_state(GRIPPER_OPEN)    # 只更新软件状态，不发硬件
robot.set_gripper_state(GRIPPER_CLOSED)
```

## 原始串口控制模板

仅在没有 GUI / 没有 `DummyRobot` 高级接口时使用。

安全打开模板：

```python
try:
    send("!HAND_DIS")
    send("!HAND_I 0.70")
    send("!HAND_EN")
    send("!HAND_O")
    time.sleep(1.00)
finally:
    send("!HAND_DIS")
    send("!HAND_DIS")
```

安全夹紧模板：

```python
try:
    send("!HAND_DIS")
    send("!HAND_I 0.70")
    send("!HAND_EN")
    send("!HAND_C")
    time.sleep(0.50)

    # 可选：夹紧后低电流保持
    send("!HAND_I 0.25")
    send("!HAND_C")
finally:
    # 如果不需要保持，必须断使能。
    # 如果启用了保持，也必须由上层定时器在 max_hold_s 后断使能。
    pass
```

如果不是明确需要夹持保持，结尾应双 `!HAND_DIS`。

相关文件：

```text

```

导出数据中的状态向量：

```text
observation.state = [x_mm, y_mm, z_mm, rot6d_0..rot6d_5, gripper]
action            = [x_mm, y_mm, z_mm, rot6d_0..rot6d_5, gripper]
```

长度为 10：

```text
3 xyz + 6 rot6d + 1 gripper = 10
```

夹爪字段语义：

```text
gripper = 0.0  # 夹紧 / 闭合
gripper = 1.0  # 打开
```

兼容旧数据：

- 旧 GUI 的 `0-30` 角度值会被归一化。
- 大于 `1.0` 的旧值中，`>=15` 会视为打开，`<15` 会视为夹紧。

AI 训练或推理时应输出二值夹爪命令：

```python
target_gripper = 1.0 if policy_wants_open else 0.0
robot.set_gripper_state(target_gripper)
```

不要让策略连续高频重复发送同一个夹爪状态。软件回放逻辑会在状态变化时才下发夹爪动作；夹紧保持期间只会按保持刷新节奏处理。

## 后续 AI 控制夹爪的规则

如果 AI 需要控制真实夹爪，应遵守以下顺序：

1. 确认机械臂串口已连接。
2. 确认夹爪硬件接线、拨码和机构装配无异常。
3. 关闭 GUI/Robot 的夹爪占位模式。
4. 使用 `robot.set_gripper_state(1.0)` 打开，或 `robot.set_gripper_state(0.0)` 夹紧。
5. 不要直接持续发送 `!HAND_O` / `!HAND_C`。
6. 需要保持夹持时使用低保持电流，并设置最大保持时长。
7. 任务结束、急停、断开连接前调用 `robot.disable_gripper()`。

AI 不应该做的事：

- 不要调用 `!HAND_ZERO` 当作自动校准。
- 不要使用 `!HAND_POS` 作为当前主控逻辑，除非重新确认位置模式和限位。
- 不要对 node 7 使用通用关节调参/重启命令。
- 不要把夹爪当第 7 关节写进 6 轴关节数组。
- 不要在没有超时保护的情况下输出持续电流。

## 安全恢复

如果怀疑夹爪仍在发力，优先发送：

```text
!HAND_DIS
!HAND_DIS
```

在 GUI 中点击：

```text
线轨夹爪控制 -> 断使能
```

在代码中调用：

```python
robot.disable_gripper()
```

如果是全臂急停，当前软件的 `robot.emergency_stop()` 会发送机械臂 `!STOP`，并额外调用夹爪断使能。
